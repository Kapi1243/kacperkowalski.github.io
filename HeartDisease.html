Heart Disease Prediction AI Model

About the Project

Technologies Used: Python, Pandas, Scikit-Learn, Decision Trees, Data Visualization (Matplotlib, Seaborn)

View Code on GitHub

Key Results:

  • Model Accuracy: 88.2% on test dataset (445,132 samples)
  • AUC-ROC Score: 0.946 (94.6% area under curve)
  • Precision: 89.9% - High confidence in positive predictions
  • Recall: 86.1% - Successfully identified 86.1% of heart disease cases
  • F1-Score: 0.88 - Strong balance between precision and recall
  • Dataset: 445,132 patient samples with 40 clinical features
  • Best Model: Random Forest Classifier outperformed Logistic Regression and XGBoost

Project Overview:

This project demonstrates the application of machine learning to predict the likelihood of heart disease in individuals based on various health-related features. Using a dataset containing patient information, such as age, cholesterol levels, blood pressure, and family history, I trained a classification model to identify patterns and predict the risk of heart disease. After pre-processing the data, including handling missing values and scaling features, I used algorithms like Decision Trees and Logistic Regression to build an accurate model. The project focused on not only achieving high accuracy but also on providing explainable results, allowing healthcare professionals to interpret the model's predictions. The final model can be used to assess an individual's risk, providing valuable insights for early diagnosis and prevention. This project showcases the potential of machine learning in healthcare, where predictive models can assist in saving lives by offering timely interventions.

Technical Details:

  • Data Preprocessing: Applied ColumnTransformer with two pipelines:
    • Numerical features: SimpleImputer (mean strategy) → Standardization
    • Categorical features: SimpleImputer (most frequent) → One-hot encoding
  • Class Balancing: Applied SMOTE (Synthetic Minority Over-sampling Technique) after preprocessing to handle class imbalance
  • Feature Selection: Focused on clinically relevant features including:
    • Patient demographics (Age, Sex, BMI)
    • Risk factors (SmokerStatus, AlcoholDrinkers, PhysicalActivity)
    • Health indicators (GeneralHealth, DiabetesStatus, KidneyDisease)
    • Medical history (HadHeartAttack, HadStroke, HadAngina)
  • Model Comparison: Evaluated 3 algorithms:
    • Logistic Regression: Baseline model with built-in regularization
    • Random Forest: Best performer (88.2% accuracy)
    • XGBoost: Gradient boosting ensemble method
  • Validation: Train-test split (80/20) with comprehensive metrics:
    • Accuracy, Precision, Recall, F1-Score
    • ROC-AUC curve analysis
    • Confusion matrices for error analysis
    • Execution time monitoring
  • Visualization: Created comprehensive analysis plots:
    • Age distribution analysis across disease/healthy groups
    • BMI correlation with heart disease risk
    • Model performance comparison charts
    • Confusion matrices for each algorithm

Challenges & Solutions:

  • Class Imbalance: Dataset showed uneven distribution between healthy and disease cases - Solved by applying SMOTE after preprocessing to generate synthetic samples of the minority class, improving model's ability to detect disease cases
  • Large Dataset Processing: 445K+ samples required efficient processing - Solved by implementing preprocessing pipeline with ColumnTransformer, allowing parallel processing of numerical and categorical features
  • Overfitting Prevention: Initial models showed potential overfitting - Mitigated by:
    • 80/20 train-test split for validation
    • Logistic Regression's built-in L2 regularization
    • Random Forest's ensemble approach (multiple decision trees)
    • Monitoring train vs test performance metrics
  • Feature Selection: 40 features required careful selection - Prioritized clinically significant features (SmokerStatus, Age, BMI, DiabetesStatus) based on medical literature and feature importance analysis from Random Forest
  • Model Interpretability: Healthcare applications require explainable predictions - Generated confusion matrices and visualization plots (Age, BMI distributions) to help medical professionals understand model decisions and risk factors
  • Missing Data: Real-world medical data contained missing values - Handled through SimpleImputer with mean imputation for numerical features and mode imputation for categorical features
