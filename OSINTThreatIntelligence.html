OSINT Threat Intelligence Pipeline

End-to-end AI system for public security-feed monitoring, threat classification, entity extraction, risk scoring, and analyst review.

Short Overview

I built an end-to-end OSINT threat-intelligence prototype that collects public security articles, cleans and deduplicates them, classifies each item into threat categories, extracts entities such as CVEs and organisations, and assigns a configurable 0-100 risk score. The system includes a Streamlit dashboard for analyst review, showing high-risk items, CVE/entity trends, article-level explanations, source reliability, and manual override controls.

Why I Built It

I created this project to explore how AI can support threat intelligence workflows while keeping human analysts in control. The goal was not to replace expert judgement, but to reduce manual monitoring effort, prioritise relevant risks, and make model outputs more explainable and auditable.

What It Does

  • Ingests public security feeds
  • Cleans, normalises, and deduplicates articles
  • Stores processed items in SQLite
  • Classifies articles into threat categories using NLP
  • Extracts CVEs, organisations, and other entities
  • Produces analyst-facing explanations using top TF-IDF feature drivers
  • Calculates a 0-100 risk score using severity indicators, confidence, source reliability, and recency
  • Flags low-confidence articles for human review
  • Displays trends and risk insights in a Streamlit dashboard
  • Supports manual analyst override
  • Can run once or on a schedule

Tech Stack

Python, Streamlit, SQLite, Scikit-learn, Pandas, TF-IDF, NLP, RSS/public security feeds

Optional/Extended Components: spaCy, Hugging Face, BeautifulSoup, APScheduler, Plotly

Architecture

Pipeline flow:

Public Security Feeds -> Ingestion -> Cleaning & Deduplication -> SQLite Storage -> NLP Classification -> Entity Extraction -> Risk Scoring -> Streamlit Dashboard -> Human Review

Responsible AI

The system is designed as a decision-support tool rather than an autonomous judgement system. Low-confidence classifications are flagged for human review, and analysts can override model outputs. The project uses public sources only, includes explainability features, and acknowledges that automated threat classification can produce false positives, source bias, and incomplete context.

Limitations

Current limitations include limited labelled evaluation data, heuristic source reliability scoring, and dependence on the quality and coverage of public feeds. Future improvements would include a larger labelled test set, improved entity linking, calibrated risk scoring, role-based dashboard access, and integration with more diverse OSINT sources.
